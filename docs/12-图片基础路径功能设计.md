# 12-图片基础路径功能设计

> 时间：2026-03-30

## 需求背景

JSON 数据中图片字段通常只存储相对路径或文件名（如 `1.png`），而实际图片存储在服务器的特定目录下。需要一个配置项将两者关联起来。

## 设计方案

### 1. 数据流

```
JSON数据: "1.png"
         ↓
项目配置 imageBasePath: "/mnt/data/images"
         ↓
实际文件路径: "/mnt/data/images/1.png"
         ↓
后端图片代理: GET /api/v1/images/{project_id}/{image_path}
         ↓
前端 img src: "/api/v1/images/1/1.png"
```

### 2. 数据库变更

**projects 表新增字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| image_base_path | VARCHAR(500) | 图片根目录路径（可选） |

### 3. 后端接口

#### 3.1 图片代理接口

```
GET /api/v1/images/{project_id}/{image_path}
```

- 根据 `project.image_base_path + image_path` 读取文件
- 支持路径中包含子目录（如 `subdir/1.png`）
- 返回图片文件流

#### 3.2 项目接口变更

**创建项目**：
```json
{
  "name": "项目名称",
  "description": "描述",
  "image_base_path": "/mnt/data/images"
}
```

**更新项目**：
```json
{
  "name": "新名称",
  "image_base_path": "/mnt/data/images/new"
}
```

### 4. 前端变更

#### 4.1 新建项目页面

- 添加"图片根目录"输入框
- 非必填，placeholder 提示示例路径

#### 4.2 项目详情页面

- 基本信息区域展示图片根目录
- 添加"编辑"按钮，支持修改项目基本信息（名称、描述、图片根目录）
- 编辑后保存到后端

#### 4.3 ImageBBoxAnnotator 组件

- 获取项目的 `image_base_path`
- 图片 URL 拼接逻辑：
  ```
  if (image_base_path) {
    imageUrl = `/api/v1/images/${projectId}/${cleanedImagePath}`
  } else {
    imageUrl = cleanedImagePath  // 原有逻辑
  }
  ```

### 5. 实现步骤

1. [ ] 后端：数据库增加 image_base_path 字段
2. [ ] 后端：项目创建/更新接口支持 image_base_path
3. [ ] 后端：实现图片代理接口
4. [ ] 前端：新建项目页面添加图片根目录输入
5. [ ] 前端：项目详情页展示并支持编辑图片根目录
6. [ ] 前端：ImageBBoxAnnotator 使用代理 URL
