# 低代码标注配置系统

## 5.1 设计理念

### 问题

传统方案中，每个新的标注任务都需要前端编写新的页面组件：
- 目标检测：需要 `<ObjectDetection.vue>`
- 图像分类：需要 `<ImageClassification.vue>`
- 文本NER：需要 `<TextNer.vue>`
- ...每增加一种标注类型，就要加一个组件

### 解决方案

定义标准化的**标注配置JSON Schema**，标注页面根据配置动态渲染组件：
- 组件是预置的，只增不减
- 配置是声明式的，描述"用什么组件"
- 同一套代码，支持无限种标注任务

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| version | string | 是 | 配置版本，用于兼容 |
| dataType | string | 是 | 数据类型：image/text |
| settings | object | 否 | 页面设置 |
| components | array | 是 | 标注组件列表 |

### settings 字段说明

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| showThumbnail | boolean | true | 是否显示缩略图 |
| enableShortcut | boolean | true | 是否启用快捷键 |
| autoSave | boolean | true | 是否自动保存 |
| autoSaveInterval | number | 30000 | 自动保存间隔(毫秒) |

### component 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 组件唯一标识 |
| type | string | 是 | 组件类型 |
| label | string | 是 | 显示标签 |
| required | boolean | 否 | 是否必填 |
| visible | boolean | 否 | 是否可见 |
| attributes | object | 否 | 组件属性 |
| validation | object | 否 | 校验规则 |

## 5.5 动态渲染原理

### 5.5.1 前端架构

```
AnnotationRunner.vue (标注执行页面)
    │
    ├── 1. 获取任务数据 (API: GET /api/tasks/{id})
    │       │
    │       └── 返回: { data_source, project: { config_json } }
    │
    ├── 2. 解析配置
    │       │
    │       └── config_json.components 数组
    │
    ├── 3. 动态渲染组件
    │       │
    │       └── <component :is="getComponent(type)" v-bind="componentConfig" />
    │
    └── 4. 收集结果
            │
            └── result_json = { componentId: value, ... }
```

### 5.5.2 组件注册

```typescript
// components/AnnotationComponentMap.ts
import ImageRect from './ImageRect.vue'
import ImageClassify from './ImageClassify.vue'
import ImagePolygon from './ImagePolygon.vue'
import TextClassify from './TextClassify.vue'
import TextNer from './TextNer.vue'
import TextInput from './TextInput.vue'

export const AnnotationComponentMap: Record<string, any> = {
  image_rect: ImageRect,
  image_classify: ImageClassify,
  image_polygon: ImagePolygon,
  text_classify: TextClassify,
  text_ner: TextNer,
  text_input: TextInput
}

export function getComponent(type: string) {
  return AnnotationComponentMap[type] || null
}
```

### 5.5.3 动态渲染实现

```vue
<!-- AnnotationRunner.vue -->
<template>
  <div class="annotation-runner">
    <!-- 数据展示区 -->
    <div class="data-display">
      <ImagePreview v-if="dataType === 'image'" :src="dataSource.url" />
      <TextDisplay v-else-if="dataType === 'text'" :content="dataSource.content" />
    </div>

    <!-- 标注组件区 -->
    <div class="annotation-panel">
      <el-form :model="formData" :rules="rules" ref="formRef">
        <template v-for="component in components" :key="component.id">
          <component
            :is="getComponent(component.type)"
            v-model="formData[component.id]"
            :label="component.label"
            :required="component.required"
            :attributes="component.attributes"
            :validation="component.validation"
          />
        </template>
      </el-form>
    </div>

    <!-- 工具栏 -->
    <div class="toolbar">
      <el-button @click="handlePrev" :disabled="!hasPrev">上一条</el-button>
      <el-button @click="handleSave" type="primary">保存</el-button>
      <el-button @click="handleNext" :disabled="!hasNext">下一条</el-button>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'
import { useRoute } from 'vue-router'
import { getComponent } from './components/AnnotationComponentMap'

const route = useRoute()
const taskId = computed(() => route.params.id)

// 数据
const configJson = ref({} as any)
const dataSource = ref({} as any)
const formData = ref<Record<string, any>>({})
const formRef = ref()

// 从配置中提取信息
const dataType = computed(() => configJson.value.dataType)
const components = computed(() => configJson.value.components || [])

// 加载任务
onMounted(async () => {
  const task = await fetchTask(taskId.value)
  dataSource.value = task.data_source
  configJson.value = task.project.config_json

  // 如果有历史结果，加载结果
  if (task.annotation) {
    formData.value = task.annotation.result_json
  } else {
    // 初始化默认值
    components.value.forEach((c: any) => {
      formData.value[c.id] = c.attributes?.defaultValue ?? null
    })
  }
})

// 保存
const handleSave = async () => {
  await formRef.value?.validate()
  await saveAnnotation(taskId.value, formData.value)
}
</script>
```

## 5.7 扩展新组件

### 添加新组件的步骤

1. **创建Vue组件**：在 `components/` 下创建组件文件
2. **注册到Map**：在 `AnnotationComponentMap` 中添加映射
3. **定义配置Schema**：在组件中定义支持的attributes
4. **处理结果格式**：定义组件value的数据结构

### 示例：添加image_point组件

```vue
<!-- components/ImagePoint.vue -->
<template>
  <el-form-item :label="label" :required="required">
    <div class="image-point-canvas" ref="canvasRef" @click="handleClick">
      <img :src="src" class="point-image" />
      <div
        v-for="(point, index) in points"
        :key="index"
        class="point-marker"
        :style="{ left: point.x + 'px', top: point.y + 'px' }"
        @click.stop="handlePointClick(index)"
      />
    </div>
    <div class="point-actions">
      <el-button size="small" @click="points.pop()">撤销</el-button>
      <el-button size="small" @click="points = []">清空</el-button>
    </div>
  </el-form-item>
</template>

<script setup lang="ts">
const props = defineProps<{
  modelValue: Array<{x: number, y: number, label?: string}>
  label: string
  required: boolean
  attributes: {
    color: string
    size: number
  }
}>()

const emit = defineEmits(['update:modelValue'])

const points = computed({
  get: () => props.modelValue || [],
  set: (val) => emit('update:modelValue', val)
})

const handleClick = (e: MouseEvent) => {
  const rect = (e.target as HTMLElement).getBoundingClientRect()
  points.value = [...points.value, {
    x: e.clientX - rect.left,
    y: e.clientY - rect.top
  }]
}
</script>
```

## 5.8 配置版本管理

### 版本演进策略

- `version` 字段标识配置版本
- 前端解析时根据版本做兼容处理
- 后端存储时不验证版本，仅作为参考

### 升级注意事项

- 新增组件类型需保持向后兼容
- 移除组件类型需做好数据迁移
- 字段变更需提供默认值
