# 项目概述与技术架构

## 1.1 项目背景

### 项目目标

搭建一个支持**多模态混合标注**的低代码平台，核心目标是：

1. **低代码配置**：通过JSON配置即可创建新的标注任务页面，无需每次新建任务时编写新代码
2. **多模态支持**：同时支持图像标注（框选、分类、OCR）和文本标注（分类、NER、问答）
3. **轻量级部署**：适配小型团队（<10人）使用

### 业务场景

- 目标检测数据集标注
- 图像分类标签标注
- 文本实体识别标注
- 文本分类标注
- 其他自定义标注任务

## 1.4 核心设计理念

### 低代码标注配置

**问题**：传统方式下，每个新的标注任务都需要前端编写新的页面组件，开发成本高。

**解决方案**：
- 定义标准化的**标注配置JSON Schema**
- 标注页面根据配置动态渲染组件
- 支持的组件类型在系统中预置，配置时引用即可

### 数据流

```
创建项目 → 配置标注组件 → 导入数据 → 分配任务 → 标注执行 → 结果存储
```

1. **管理员**创建项目，配置标注页面（选择组件、设置属性）
2. **管理员**导入待标注数据（图片URL/路径、文本内容）
3. **系统**自动创建任务，分发给标注员
4. **标注员**打开标注页面，根据配置动态渲染的组件进行标注
5. **标注完成**后，结果JSON存入数据库

## 1.5 非功能性需求

### 性能

- 单用户标注响应时间 < 200ms
- 支持10人同时在线标注

### 安全

- JWT Token认证
- 密码bcrypt加密存储
- 接口权限校验

### 可扩展性

- 组件系统支持后续添加新的标注组件
- 配置JSON支持版本演进
