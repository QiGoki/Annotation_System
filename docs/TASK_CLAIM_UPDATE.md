# 任务领取机制 - 更新说明

## 更新时间
2026-03-24

## 变更内容

### 1. 项目管理页增强

**新增功能**：
- 成员管理卡片：显示项目所有成员
- 添加成员按钮：打开对话框选择用户并设置角色
- 成员角色编辑：下拉框直接切换管理员/成员角色
- 移除成员：删除项目成员资格

**访问路径**：
- 项目列表 → 点击项目详情 → 成员管理区域

### 2. 任务列表页重构

**删除页面**：
- `MyTasks.vue`（我的任务页面已删除）

**保留页面**：
- `TaskList.vue`（任务列表页面）

**新增筛选系统**：
| 筛选项 | 说明 |
|--------|------|
| 所属项目 | 下拉选择，动态加载用户参与的项目 |
| 状态 | 待领取/标注中/已完成 |
| 领取人 | 下拉选择项目成员 |

**任务列表字段**：
| 列名 | 说明 |
|------|------|
| ID | 任务编号 |
| 所属项目 | 任务归属的项目名称 |
| 数据源 | 图像/文本内容预览 |
| 状态 | 任务当前状态 |
| 领取人 | 当前领取任务的用户 |
| 创建时间 | 任务创建时间 |
| 操作 | 领取/释放/标注按钮 |

### 3. 路由调整

**删除路由**：
- `/my-tasks` → 已移除

**保留路由**：
- `/tasks` → 任务列表（带筛选）

**导航栏**：
- 项目管理
- 任务列表
- 用户管理（仅管理员）

## 后端 API

### 新增端点
| 端点 | 功能 |
|------|------|
| `GET /api/v1/project-members/my-projects` | 我的项目列表 |
| `GET /api/v1/project-members/{id}/members` | 项目成员列表 |
| `POST /api/v1/project-members/{id}/members` | 添加成员 |
| `DELETE /api/v1/project-members/{id}/members/{uid}` | 移除成员 |
| `PUT /api/v1/project-members/{id}/members/{uid}/role` | 更新角色 |
| `POST /api/v1/tasks/{id}/claim` | 领取任务 |
| `POST /api/v1/tasks/{id}/release` | 释放任务 |

### 任务列表 API 参数
```
GET /api/v1/tasks?project_id=1&status=pending&assigned_to=1
```

| 参数 | 类型 | 说明 |
|------|------|------|
| project_id | int | 按项目筛选 |
| status | string | 按状态筛选 (pending/doing/completed) |
| assigned_to | int | 按领取人筛选 |
| page | int | 页码（默认 1） |
| page_size | int | 每页数量（默认 100） |

## 使用流程

### 项目成员管理
1. 访问项目详情页面
2. 点击"添加成员"按钮
3. 选择用户和角色
4. 确认添加

### 任务领取
1. 访问任务列表页面
2. 筛选所属项目（可选）
3. 找到待领取任务（状态：待领取）
4. 点击"领取"按钮
5. 任务状态变为"标注中"

### 任务标注
1. 在任务列表中找到已领取任务
2. 点击"标注"按钮
3. 在新窗口打开标注页面
4. 完成标注后保存/提交

### 任务释放
1. 找到已领取的任务（自己或其他管理员）
2. 点击"释放"按钮
3. 确认后任务状态变回"待领取"
