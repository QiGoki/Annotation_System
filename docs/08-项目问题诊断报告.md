# 项目问题全面诊断报告

> 诊断时间：2026-03-28

## 一、API 路由不匹配问题（严重）

### 1.1 前后端 API 路径不匹配

| 前端调用 | 后端实际路径 | 问题 |
|---------|-------------|------|
| `/export/import` | `/api/v1/export/import` | 前端 baseURL 已包含 `/api/v1`，路径正确 |
| `/project-members/my-projects` | 路由注册顺序问题 | ⚠️ 路由冲突 |

### 1.2 路由冲突：project_members.py

```python
# 问题代码结构：
@router.get("/{project_id}/members")     # 路径参数优先匹配
@router.get("/my-projects")               # 永远不会被匹配到！
```

**原因**：`/{project_id}/members` 会把 `my-projects` 当作 `project_id` 参数匹配。

**解决方案**：把 `/my-projects` 路由移到 `/{project_id}/members` 之前，或者改为 `/projects/my-projects`。

### 1.3 前端 API 重复定义

**问题文件**：`frontend/src/api/project_members.ts` 和 `frontend/src/api/task.ts`

```typescript
// project_members.ts 中重复定义
export function claimTask(taskId: number) { ... }
export function releaseTask(taskId: number) { ... }

// task.ts 中也定义了相同的函数
export function claimTask(taskId: number) { ... }
export function releaseTask(taskId: number) { ... }
```

## 三、路由问题

### 3.1 路由守卫逻辑问题

**文件**：`frontend/src/router/index.ts`

```typescript
// 问题：根路径重定向依赖 userStore.user，但此时可能还没加载
redirect: () => {
  const userStore = useUserStore()
  return userStore.user?.role === 'admin' ? '/projects' : '/tasks'
}
```

**问题**：首次访问时 `userStore.user` 可能为 null，导致重定向逻辑不确定。

### 3.2 标注页面布局嵌套

```typescript
// 路由配置
{
  path: '/annotate/:id',
  component: AnnotationLayout,
  children: [{
    path: '',
    component: AnnotationRunner
  }]
}
```

**问题**：`AnnotationLayout` 在 `onMounted` 中检查登录状态是异步的，可能导致闪烁。

## 四、UI 样式问题

### 4.1 固定高度/宽度

**问题文件**：多个组件

```scss
// Layout.vue
.layout-container { height: 100vh; }  // OK

// ProjectCreate.vue
.project-create { height: 100vh; }    // 问题：嵌套在 Layout 中，会超出

// ProjectConfigurator.vue
.canvas { height: 500px; }            // 问题：固定高度，不自适应
```

### 4.2 缺少响应式设计

| 组件 | 问题 |
|------|------|
| `Layout.vue` | 侧边栏固定 220px，小屏幕无适配 |
| `ProjectConfigurator.vue` | grid 固定 `1fr 350px`，小屏幕会挤压 |
| `AnnotationRunner.vue` | 三列布局无响应式 |
| `ImageBBoxAnnotator.vue` | 属性面板固定 280px |

### 4.3 CSS 变量未统一

项目中使用了多种 CSS 变量前缀：
- `--bili-*` (B站风格)
- Element Plus 默认变量

部分组件未定义 fallback 值。
