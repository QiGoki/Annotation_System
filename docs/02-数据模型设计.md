# 数据模型设计

## 2.1 数据库表总览

| 表名 | 描述 | 主要字段 |
|------|------|---------|
| users | 用户表 | id, username, password_hash, role, created_at |
| projects | 项目表 | id, name, description, config_json, created_by, created_at |
| tasks | 任务表 | id, project_id, data_source, status, assigned_to, created_at |
| annotations | 标注结果表 | id, task_id, result_json, created_at, updated_at |

---

## 2.2 用户表 (users)

### 用途
存储系统用户信息，包括管理员和标注员。

### 表结构

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) NOT NULL DEFAULT 'annotator',  -- 'admin' | 'annotator'
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_role ON users(role);
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | SERIAL | 是 | 主键，自增 |
| username | VARCHAR(50) | 是 | 用户名，唯一 |
| password_hash | VARCHAR(255) | 是 | bcrypt加密后的密码 |
| role | VARCHAR(20) | 是 | 角色：admin-管理员，annotator-标注员 |
| is_active | BOOLEAN | 否 | 账号是否启用 |
| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |

## 2.3 项目表 (projects)

### 用途
存储标注项目配置，包括项目名称、描述、标注页面配置JSON。

### 表结构

```sql
CREATE TABLE projects (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    config_json JSONB NOT NULL,  -- 标注组件配置
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT FALSE
);

CREATE INDEX idx_projects_creator ON projects(created_by);
CREATE INDEX idx_projects_name ON projects(name);
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | SERIAL | 是 | 主键 |
| name | VARCHAR(100) | 是 | 项目名称 |
| description | TEXT | 否 | 项目描述 |
| config_json | JSONB | 是 | 标注配置，定义页面有哪些组件 |
| created_by | INTEGER | 是 | 创建者ID，外键关联users |
| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |
| is_deleted | BOOLEAN | 否 | 软删除标记 |

### config_json 结构示例

```json
{
  "version": "1.0",
  "dataType": "image",
  "components": [
    {
      "id": "rect_1",
      "type": "image_rect",
      "label": "标注目标",
      "required": true,
      "attributes": {
        "minPoints": 1,
        "maxPoints": 10,
        "color": "#ff0000"
      }
    },
    {
      "id": "classify_1",
      "type": "image_classify",
      "label": "目标类别",
      "required": true,
      "attributes": {
        "options": ["人", "车", "动物", "其他"],
        "multi": false
      }
    },
    {
      "id": "input_1",
      "type": "text_input",
      "label": "备注",
      "required": false,
      "attributes": {
        "placeholder": "请输入备注信息",
        "maxLength": 500
      }
    }
  ]
}
```

## 2.4 任务表 (tasks)

### 用途
存储具体的标注任务，每个任务对应一条待标注的数据。

### 表结构

```sql
CREATE TABLE tasks (
    id SERIAL PRIMARY KEY,
    project_id INTEGER NOT NULL REFERENCES projects(id),
    data_source JSONB NOT NULL,  -- 数据源：图片URL/路径 或 文本内容
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    assigned_to INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

CREATE INDEX idx_tasks_project ON tasks(project_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_assignee ON tasks(assigned_to);
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | SERIAL | 是 | 主键 |
| project_id | INTEGER | 是 | 所属项目ID，外键 |
| data_source | JSONB | 是 | 数据源内容 |
| status | VARCHAR(20) | 是 | 任务状态：pending/doing/completed |
| assigned_to | INTEGER | 否 | 分配给的用户ID |
| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |
| completed_at | TIMESTAMP | 否 | 完成时间 |

### data_source 结构示例

**图像数据**：
```json
{
  "type": "image",
  "url": "/data/images/001.jpg",
  "width": 1920,
  "height": 1080,
  "filename": "001.jpg"
}
```

**文本数据**：
```json
{
  "type": "text",
  "content": "今天天气很好，我们去公园玩吧。"
}
```

## 2.5 标注结果表 (annotations)

### 用途
存储每条任务的标注结果。

### 表结构

```sql
CREATE TABLE annotations (
    id SERIAL PRIMARY KEY,
    task_id INTEGER NOT NULL UNIQUE REFERENCES tasks(id),
    result_json JSONB NOT NULL,  -- 标注结果
    created_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_annotations_task ON annotations(task_id);
CREATE INDEX idx_annotations_creator ON annotations(created_by);
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | SERIAL | 是 | 主键 |
| task_id | INTEGER | 是 | 任务ID，外键，唯一 |
| result_json | JSONB | 是 | 标注结果JSON |
| created_by | INTEGER | 是 | 标注人ID |
| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |

### result_json 结构示例

```json
{
  "rect_1": [
    {"x": 100, "y": 200, "width": 150, "height": 80},
    {"x": 300, "y": 400, "width": 100, "height": 60}
  ],
  "classify_1": "车",
  "input_1": "目标清晰，易于识别"
}
```

## 2.7 索引设计

| 索引名 | 表 | 字段 | 用途 |
|--------|-----|------|------|
| idx_users_username | users | username | 登录查询 |
| idx_users_role | users | role | 按角色筛选用户 |
| idx_projects_creator | projects | created_by | 查询用户创建的项目 |
| idx_projects_name | projects | name | 项目名称搜索 |
| idx_tasks_project | tasks | project_id | 查询项目下的任务 |
| idx_tasks_status | tasks | status | 按状态筛选任务 |
| idx_tasks_assignee | tasks | assigned_to | 查询用户待办任务 |
| idx_annotations_task | annotations | task_id | 查询任务标注结果 |
| idx_annotations_creator | annotations | created_by | 查询用户标注历史 |

## 2.8 存储策略建议

### 标注结果存储

- **当前设计**：JSONB存入数据库
- **适用场景**：单条结果 < 1MB，并发量 < 50人
- **如需优化**：
  - 大JSON（>1MB）可存OSS，数据库存URL
  - 高并发时可考虑MongoDB存储JSON

### 图片存储

- 当前设计：服务器本地路径挂载
- 图片不存入数据库，仅存储路径引用
- 后续可扩展为OSS存储
